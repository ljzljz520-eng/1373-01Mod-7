# 图书管理系统 (Book Management System)

## 技术栈

- **Frontend**: HTML5 + CSS3（不使用 JavaScript）
- **Backend**: 无（纯静态页面）
- **规范**: 相对路径、英文文件名，详细约束见根目录 `DESIGN.md` / `user_rule.md`

---

## 功能限制说明

题目限制不允许使用 JavaScript 或任何后端逻辑，因此本项目仅实现**静态页面与业务流程展示**：  
- 登录、注册、借阅、续借、归还等操作仅通过表单跳转和示例按钮模拟，不做真实身份校验或数据持久化；  
- 借阅记录、读者信息、统计数据均为写死在 HTML 中的示例数据，刷新页面后不会发生变化。  
如需真实业务功能，需要在本项目基础上另外引入前端脚本和后端服务，这不在当前作业要求之内。

---

## 启动方式

本项目为纯静态网站，**不再依赖 Docker**。推荐直接在本地使用浏览器打开 HTML 文件进行预览：

1. 在文件管理器中打开本项目目录 `label-1373/frontend/`。  
2. 双击 `index.html`，或右键选择使用浏览器打开。  
3. 通过页面中的导航链接浏览登录、注册、图书列表、借阅说明、我的借阅、后台管理等页面。

> 提示：在 `file://` 本地打开模式下，表单的 `method="post"` 只是结构示意，浏览器不会真正向服务器发送 POST 请求，页面之间的跳转只是普通的本地文件切换。

## 说明与演示路径

- 系统为纯前端静态实现：所有页面仅依赖 HTML 与 CSS，不使用 JavaScript 或后端服务。
- 登录页为静态登录示例：使用 POST 表单和两个提交按钮，分别示意跳转到“我的借阅”（borrow-user.html）或“后台管理”（manage.html），不做真实身份校验或数据处理。
- 图书列表卡片含“借阅（示例）”按钮，可跳转到“我的借阅”查看在借/历史示例。
- “我的借阅”页中的“续借（示例）”“归还（示例）”按钮采用纯 CSS 方式：点击后在同一行显示文字提示，仅用于说明真实系统中会提交续借/归还请求，不跳转页面、不修改数据。
- 后台借阅管理、读者管理、统计概览均为示例数据，仅作展示。
- 所有封面图片已使用有损压缩与尺寸限制优化（单张不超过约 500KB），适合直接用于静态托管与打包分发。

## 样式异常排查

如出现“页面结构在但样式偶发失效”的情况，可按以下顺序排查：

1. 强制刷新浏览器缓存（`Ctrl+F5` 或 `Cmd+Shift+R`）。
2. 确认是从 `frontend/index.html` 入口打开，避免在更深层目录直接打开导致相对路径异常。
3. 检查页面源码中 CSS 链接是否为 `css/style.css?v=20260301`（已加版本参数用于规避旧缓存）。
